# 🛡️ SIEM Lite

**A real-time Security Information & Event Management system built for learning, demos, and portfolio projects.**

SIEM Lite simulates a fully operational SOC (Security Operations Center) environment — generating live security events, detecting threats using custom rules, and presenting everything through a professional dark-themed dashboard.

---

## ⚡ Key Features

### 🔴 Real Time Everything
- **Live event stream** via WebSocket — events generated every 1–4 seconds
- **Real-time detection engine** with 12 built-in rules (brute force, ransomware, DNS tunneling, etc.)
- **Live alert notifications** with bell icon, severity-coded badges, and sound
- **Events-per-second (EPS) counter** displayed in the sidebar

### 📊 11 Interactive Dashboard Pages

| Page | Description |
|------|-------------|
| **Dashboard** | Real-time overview with event counters, severity breakdown, live event ticker, and trend charts |
| **Live Events** | Streaming event feed with filters by type, severity, IP, and protocol |
| **Alerts** | Triggered alerts from the detection engine with status management (open → investigating → resolved) |
| **Incidents** | Incident response workflow — create, assign, track, and resolve security incidents |
| **Threat Intel** | IOC database (IPs, domains, hashes, URLs, emails) with confidence scores and sources |
| **Log Search** | Full-text search across all events with advanced filters |
| **MITRE ATT&CK** | Visual MITRE ATT&CK matrix heatmap showing tactic/technique coverage |
| **Network Map** | Geo-located attack visualization — see where attacks originate in real-time |
| **Analytics** | Charts and breakdowns — severity trends, event type distribution, top attackers |
| **Reports** | Generate executive summaries and compliance reports |
| **Settings** | Manage detection rules — enable/disable, edit logic, thresholds, and time windows |

### 🧠 AI-Powered Analysis
- **Gemini AI integration** for intelligent alert analysis and recommendations
- Powered by Google's `@google/genai` SDK

### 🛡️ Real Threat Intelligence
- Automatically syncs **15,000+ malicious IPs** from real feeds on startup:
  - [CINS Army](http://cinsscore.com/list/ci-badguys.txt) — Known bad actors
  - [Feodo Tracker](https://feodotracker.abuse.ch/downloads/ipblocklist.txt) — Botnet C2 IPs
- IOC database seeded with indicators from VirusTotal, AbuseIPDB, MalwareBazaar, MISP, etc.

### 📐 Detection Engine
- **12 detection rules** covering MITRE ATT&CK tactics:
  - Brute Force, Port Scan, Malware, Data Exfiltration, Phishing, Privilege Escalation, DNS Tunneling, Lateral Movement, Suspicious PowerShell, Cloud IAM Anomaly, WAF Violations, Ransomware Behavior
- **Event correlation engine** for multi-event pattern matching
- Rules are fully configurable via the Settings UI

### 🏢 Compliance Frameworks
- Pre-loaded compliance checks for **PCI-DSS**, **HIPAA**, **SOC2**, and **NIST**
- Each control tracked with pass/fail/warning status

## 🚀 Quick Start

### Prerequisites
- **Node.js** v18+
- **npm** v9+

### 1. Install Dependencies

```bash
# Install all dependencies (server + client)
npm run setup
```

Or manually:
```bash
cd server && npm install
cd ../client && npm install
```

### 2. Seed the Database

```bash
npm run seed
```

This creates the SQLite database and seeds:
- ✅ 12 detection rules
- ✅ 15 network assets
- ✅ 15 threat intel IOCs
- ✅ 30 compliance checks
- ✅ 15,000+ malicious IPs from live threat feeds

> **Note:** No historical events are seeded — the SIEM displays **only real-time data** generated by the live simulator.

### 3. Configure Environment (Optional)

```bash
# server/.env
GEMINI_API_KEY=your_gemini_api_key    # For AI-powered analysis
PORT=3001                              # Backend port (default: 3001)
```

### 4. Start the Application

```bash
# Start both server + client concurrently
npm run dev
```

Or start them separately:
```bash
# Terminal 1 — Backend
cd server && npm start

# Terminal 2 — Frontend
cd client && npm run dev
```

### 5. Open in Browser

- 🖥️ **Dashboard**: http://localhost:5173
- ⚙️ **API**: http://localhost:3001/api
- 💚 **Health Check**: http://localhost:3001/api/health
- 🔌 **WebSocket**: ws://localhost:3001/ws

## 📡 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/health` | Server health + event/alert counts |
| GET | `/api/dashboard/stats` | Real-time dashboard statistics |
| GET | `/api/events` | List events (paginated, filterable) |
| GET | `/api/alerts` | List alerts with status filters |
| PATCH | `/api/alerts/:id` | Update alert status/assignment |
| GET | `/api/incidents` | List all incidents |
| POST | `/api/incidents` | Create new incident |
| PATCH | `/api/incidents/:id` | Update incident |
| GET | `/api/threat-intel` | List IOCs |
| GET | `/api/assets` | List network assets |
| GET | `/api/rules` | List detection rules |
| POST | `/api/rules` | Create detection rule |
| PATCH | `/api/rules/:id` | Update detection rule |
| GET | `/api/reports/:type` | Generate reports |
| POST | `/api/ai/analyze` | AI-powered alert analysis |
| WS | `/ws` | Live event + alert stream |

## 🔒 Database Schema

6 tables with full indexing:

| Table | Purpose |
|-------|---------|
| `events` | All security events (auth, firewall, IDS, malware, DLP, etc.) |
| `alerts` | Triggered detection alerts linked to events and rules |
| `incidents` | Security incident lifecycle management |
| `rules` | Detection rule definitions with MITRE mapping |
| `threat_intel` | Indicators of Compromise (IOC) database |
| `assets` | Network asset inventory |
| `compliance_checks` | PCI-DSS, HIPAA, SOC2, NIST control status |
